REpresentational State Transfer

	A kommunikáció állapotmentes. A Szerver nem tartja karban a kliensei állapotát

	Általában JSON a kommunikáció adatfomrátuma.
	
	A REST egy implementációja és használata a HTTP protokollnak.
	A REST tehát egy mód ahogy szoftverkomponensek kommunikálnag egymással a weben.
	
	Rules:
		- A metódus információkat HTTP igékkel kell kifejezni:
			Nem RESTES: GET api/users/delete/:userId HTTP/1.1
			RESTES (helyes megoldás) DELETE api/users/delete/:userId HTTP/1.1
		
		- A scope információnak az URI-ban át kell mennie:
			DELETE api/users/delete/:userId HTTP/1.1	//Itt a :userId az
		
	Diagram:
		A Kliens küld Requestként egy HTTP message-et, aminek tartalma egy JSON file, 
		specifikálja az URI-val hogy mely metódust szeretné meghívni,
		A Post, vagy PUT request tárgyát pedig tartalmazza a BODY.
		
		A szerver megkapja a Requestet, kibontja, analizája, elvégez pár mûveletet,
		majd Response--ként visszaküld egy HTTP messaget
